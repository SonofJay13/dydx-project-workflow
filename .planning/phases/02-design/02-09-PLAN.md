DESIGN-30 drift-detection contract shape:

```markdown
**Inputs.**
- Current sandbox schema (fetched fresh by pre-flight: `platform-<X>` skill calls `table_columns_read` / `GET /schema` / equivalent per platform).
- Cached `last_known_schema.<platform>.schema_hash` from `client_state.yaml`.

**Outputs.**
- **Match** (current schema_hash == cached schema_hash): proceed to test execution.
- **Mismatch**: HALT before any test runs. Emit `<Client> Brain/test-bot/schema_drift_report.md` instead of executing.

**Halt condition.**
Mismatch detected → no tests run, no sandbox writes, no `last_passed_at:` updates. Skill exits with non-zero status; `08d_test-results_v<N>.md` records the halt event in lieu of test results.

**`schema_drift_report.md` shape.**
- Frontmatter: `client:`, `platform:`, `previous_schema_hash:`, `current_schema_hash:`, `detected_at:`.
- Body: per-column diff (added / removed / type-changed columns); recommended human action (acknowledge if intended schema change → 8a runs to update `last_known_schema`; revert if unintended → fix sandbox; investigate if cause unknown).

**Human-action requirement.**
Drift requires explicit human acknowledgment OR revert before next 8d run. Skill REFUSES to run with stale `last_known_schema` (would invalidate test assumptions). Two human action paths:
1. **Acknowledge** (intended schema change): human runs `provision-test-harness` (8a) which fetches new schema, updates `last_known_schema`, marks affected TCs as `quarantined` for review.
2. **Revert** (unintended schema change): human fixes sandbox to match `last_known_schema`; reruns 8d.

**Algorithmic detail (Stage 8 build phase territory).**
Per CONTEXT D-30, this contract specifies inputs / outputs / halt condition / report shape / human-action requirement WITHOUT numbered pseudocode. The actual algorithm (column-by-column comparison logic, hash computation, report-generation templating) is implemented in v2.1+ Phase 5 Stage 8 build per CHANGE-01 9-phase plan.
```

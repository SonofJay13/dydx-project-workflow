Coda mechanics subsection shape:

```markdown
**Coda integration mechanics (DESIGN-22 contract — exact values).**

- **Schema introspection:** First Stage 6 run per client introspects the client's existing Coda task table schema (column names + types) via Coda MCP `table_columns_read`. Schema cached in `<Client> Brain/00_HUB.md` `coda_tasks_schema:` block (per Phase 4 OPEN-05 default 00_HUB.md schema). Subsequent runs read cache; cache invalidates if `mutationStatus` polling reveals schema mismatch.
- **Upsert pattern:** Coda writes via `table_rows_manage` (Coda MCP) with `keyColumns` parameter set to the client task table's primary-key column(s) (typically `task_id` OR `task_name`). Idempotent: same `keyColumns` value with new other-column values updates the existing row; new value inserts.
- **Mutation status polling:** Each `table_rows_manage` call returns a `mutationStatus` ID. Stage 6 polls `mutationStatus` until terminal state (`success` or `error`) before proceeding. Failure halts with explicit error in `06_cost_v<N>.md`; no silent ignore.
- **Rate limit:** 4 req/10 second sliding window (= 80% of Coda public ceiling 5 req/10s, leaves headroom for concurrent reads). Limiter is per-token, applied at the Stage 6 skill level. Burst handling: if approaching ceiling, skill yields with backoff before next call.
- **Wait-for-commercial-inputs gate (DESIGN-22 contract):** Stage 6 has TWO halt points before client-facing cost summary:
  1. **Pre-write halt:** Skill emits a `06_cost_inputs_v<N>.md` artefact listing all commercial inputs needed (rate per assignee class; client-specific multipliers; pre-discount adjustments). Halts. Human supplies values via frontmatter or inline. Skill resumes only when `commercial_inputs_status: provided` is set.
  2. **Pre-publish halt:** After computing `risk_adjusted_hours` and writing `06_cost_v<N>.md`, skill halts BEFORE pushing the client-facing summary to Coda. Human reviews `06_cost_v<N>.md`, sets `status: client_review` after sanity-checking. Coda push runs only on `status: approved` write.
```

> **DESIGN-17:** Stage 1 Kickoff capture — inputs (meeting notes / client requirements / internal feedback / Miro workflow map); dual artefact branching (discovery-ready vs draft SOW); Field Notes triage from Coda brain doc (defaults to `processed_at IS NULL`, never auto-merges per DESIGN-09); Miro paste fallback when API ingest unavailable; auto-classification into kickoff template sections with explicit "unknown" markers.

**Stage:** 1 (file prefix `01_kickoff_*` per DESIGN-02).
**Complexity:** Medium.

**Inputs.**
- **Frontmatter consumed:** `client:` + `project:` + (optional) `frontmatter_version: 2`. New artefact (no `based_on_*` upstream).
- **Upstream artefact paths:** N/A (Stage 1 is the entry point of the pipeline).
- **External inputs:** meeting notes (paste), client requirements docs (paste OR Drive ref), internal feedback (paste), Miro workflow map (paste — see fallback below); Field Notes from `<Client> Brain` Coda doc (read-only via Coda MCP per AUDIT.md §AUDIT-08).

**Outputs.**
- **Carrier file:** `01_kickoff_v<N>.md` in `<Client> Brain/<Project>/`.
- **Frontmatter set:**
  - `client:`, `project:`, `frontmatter_version: 2`
  - `kickoff_branch: discovery-ready | draft-sow` — locks downstream routing per DESIGN-18 / DESIGN-19
  - `field_notes_processed_count: <N>` — count of Field Notes triaged this kickoff (per `processed_at IS NULL` filter per DESIGN-09)
  - `status: draft` initially; gates on `status: approved` for Stage 2/3 entry
- **Branch routing (DESIGN-17 dual-branch contract):**
  - `kickoff_branch: discovery-ready` → carrier follows kickoff template (sections per template); Stage 2 (Discovery) consumes.
  - `kickoff_branch: draft-sow` → carrier carries enough SOW skeleton that Stage 2 SKIPS; Stage 3 (SOW) consumes directly.

**Downstream consumer.**
- discovery-intake (Stage 2) — IF `kickoff_branch: discovery-ready`.
- generate-sow (Stage 3) — IF `kickoff_branch: draft-sow` (skips Stage 2).

**Status flag(s).**
- `status: approved` on `01_kickoff_v<N>.md` gates either downstream stage.

**Hand-off message (verbatim from DESIGN-13 matrix Stage 1 → Stage 2 row).**

> Awaiting `status: approved` write to `01_kickoff_v<N>.md`. Branch routing on `kickoff_branch:` value (`discovery-ready` → Stage 2; `draft-sow` → SKIP Stage 2 → Stage 3).

**Key v2 decisions for this stage.**

1. **Dual-branch artefact** — kickoff outputs ONE artefact but it can be either a discovery-ready briefing or a draft SOW. The `kickoff_branch:` frontmatter field is the routing key. Single skill, two valid output shapes; decision is made by the kickoff author (human) at write-time.
2. **Field Notes triage filter** — `processed_at IS NULL` is the default Coda query (per DESIGN-09 directional boundary). Each note is QUOTED in the kickoff (not auto-merged); human chooses keep / drop / edit-and-keep. Notes marked keep have their Coda row's `processed_at:` updated by the skill (one-way write per DESIGN-09 Coda mirror direction). Notes marked drop have `processed_at:` set with a `drop_reason` field.
3. **Miro paste fallback** — IF Miro MCP not wired (per DESIGN-07 connector probe — currently MISSING per AUDIT.md §AUDIT-08), the skill prompts human to paste exported Miro workflow map text; the skill auto-classifies into kickoff template sections. IF Miro API later available, swimlane-reconstruction algorithm is build-phase research (per CONTEXT Deferred Ideas). Paste fallback works without Miro MCP.
4. **Auto-classification with `[unknown]` markers** — when input content does not cleanly map to a kickoff template section, the skill writes the content under the closest section with an inline `[unknown — needs human classification: <one-line summary>]` marker. Human resolves before `status: approved` write.

**Dependencies.**
- DESIGN-09 (directional boundary: Field Notes never auto-merged)
- DESIGN-07 (connector probe surfaces Miro MCP missing → paste fallback active)
- DESIGN-01 (frontmatter scheme: `kickoff_branch:` joins canonical lifecycle)

**Cross-references.** DESIGN-18 (Stage 2 reads `kickoff_branch:` to skip when `draft-sow`); DESIGN-19 (Stage 3 reads same field to consume from kickoff directly when `draft-sow`); AUDIT.md §AUDIT-08 (Miro MCP missing finding); AUDIT.md §AUDIT-01.1 (v0.3.0 discovery-intake brittleness — Field Notes auto-merge risk addressed by this contract).

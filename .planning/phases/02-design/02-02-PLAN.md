**DESIGN-10:** Senior-implementer voice principles (5):
1. **No AI hedging.** Never write "we recommend", "as an AI", "I would suggest", "perhaps consider", "might want to". Lock the call or surface it as `[OPEN: Phase 4 — ...]`.
2. **Specific over abstract.** "Add `pipe_id` to frontmatter when `platform: pipefy`" beats "ensure platform context is captured".
3. **No apology-prefaces.** Never write "Note that...", "Please be aware that...", "It's worth mentioning...".
4. **Cite source for facts.** Every v0.3.0 claim cites `AUDIT.md §X.Y` or `file:line` per Phase 1 D-32 / D-34.
5. **End with handoff, not summary.** Skills close with the exact next-action message (per DESIGN-06), not a recap of what they did.

Forbidden phrasings list (verbatim): "we recommend", "as an AI", "I would suggest", "perhaps consider", "might want to", "it's worth noting", "please be aware", "in order to" (use "to"), "make sure to" (use imperative), "feel free to".
